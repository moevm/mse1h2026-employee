# mse-template
## О проекте
Проект направлен на разработку Telegram-бота для контроля выполнения задач сотрудниками и ведения ежедневной отчётности внутри команды. Основные цели проекта – упрощение контроля работы сотрудников, повышение дисциплины и создание истории выполненных задач.

### Основные категории пользователей
Основные виды пользователей, взаимодействующие с системой: 
* Пользователь без роли;
* Стажер;
* Сотрудник;
* Руководитель;
* Супер-пользователь (обладает всем функционалом руководителя и администрирует роли других пользователей).

### Требуемый функционал проекта
Для достижения поставленных целей используется следующий основной набор подсистем:
* Модуль фиксации рабочего времени сотрудников;
* Модуль администрирующий полного жизненного цикла задачи (от создания до успешного завершения с сохранением результатов);
* Модуль генерации недельного отчета о сотруднике на основании результатов его деятельности.

#### Функционал ролей
Для каждой из описанных выше ролей необходимо реализовать следующий функционал:
* Роль Пользователь без роли:
  * Создание запроса о назначении роли;
  * Получение доступа к функционалу одной из прочих ролей при наличии соответствующих прав.
* Роль Сотрудник:
  * Внесение сведений о начале и завершении рабочего дня;
  * Получение уведомлений от бота о необходимости внесения сведений, описанных выше, если они не были отправлены;
  * Самостоятельное создание задач (они назначаются только этому пользователю и перед назначением должны быть утверждены руководителем);
  * Просмотр списка назначенных ему задач;
  * Принятие задачи к выполнению;
  * Создание отчета о выполненной задаче;
  * Просмотр комментария руководителя, если задача была отклонена.
* Роль Руководитель:
  * Создание и назначение задачи сотруднику;
  * Просмотр списка активных задач;
  * Просмотр списка отчетов, отправленных ему на проверку;
  * Подтверждение выполнения задачи;
  * Отправка задачи на доработку с комментарием;
  * Создание недельного отчета о сотруднике;
  * Кроме этого ему доступен весь функционал роли Сотрудник.
* Роль Супер-пользователь:
  * Присвоение роли пользователю;
  * Просмотр списка пользователей, ожидающих назначения роли;
  * Запрет пользователю на использование бота;
  * Кроме этого ему доступен весь функционал роли Руководитель.

Роль Стажер является аналогом роли Сотрудник, однако он не может самостоятельно создавать задачи и не имеет доступа к функционалу фиксации рабочего времени сотрудника.

#### Дополнительные сведения о функционале проекта
Задача может иметь один из четырех статусов:
* Назначена (появляется при создании задачи);
* В работе (появляется при принятии задачи к выполнению сотрудником);
* Выполнена (появляется после подтверждения отчета руководителем);
* Отклонена (появляется при отправке задачи на доработку).

Сведения о задаче сохраняются в базу данных только при получении статуса Выполнена.
Отчеты сотрудники должны отправлять в виде файлов.
